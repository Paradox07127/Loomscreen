type 加小图标(SF Symbols)

与现有拖拽图标一致:Scene=cube.transparent、Video=play.rectangle、Web=globe。极小前置字形,扫读更快,不喧宾夺主。

基础已在:AdaptiveGlass

app 的 adaptiveGlassSurface / AdaptiveGlassContainer / adaptiveGlassButton 在 macOS 26 已直接走原生 .glassEffect / .glassProminent / GlassEffectContainer,旧系统降级 material。筛选栏搜索胶囊已在用。

角标用 .glassEffect 而非黑底

评分/分辨率/In-Library 角标改成 .glassEffect(.regular, in: .capsule) 浮在壁纸上——折射 + 高光 + 环境染色,正是 Photos/TV 在 26 上的做法。这是最直接的"更液态玻璃"。

chip 合并 + 形变

把 Type chip 放进一个 GlassEffectContainer 并用 glassEffectID,相邻玻璃会流体般融合/分离(macOS 26 招牌动效),比独立胶囊更高级。

主操作用 glassProminent

Open in Steam / Apply 从 .borderedProminent 换 .glassProminent(accent 染色玻璃):仍是 royal blue,但有液态玻璃的深度与高光。Download/次级用 .glass。

克制:内容区不要全玻璃

玻璃只用于 chrome / 控件 / 角标。hero 大图、标题、描述这类内容仍用实底,保证可读性(对比度 ≥ 4.5:1),并尊重"降低透明度"无障碍设置——这点 AdaptiveGlass 已处理。

弹簧动效 + 环境响应

inspector 进出、chip 切换用 spring;玻璃染色随下方壁纸主色微移(原生 glass 自带)。reduce-motion 时退化为纯透明度过渡。

渲染为静态期望稿,用于评审与对齐;颜色/间距/圆角取自 app 的 DesignTokens 与 AccentColor。Liquid Glass 部分为 CSS 近似(backdrop-filter + 高光层),实际由 SwiftUI 原生 .glassEffect 渲染,质感更佳。
